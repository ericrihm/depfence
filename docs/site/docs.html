Installation

depfence requires Python 3.10 or later. It has no mandatory compiled dependencies — pure Python wheels install everywhere.

pip (recommended)

bash
pip install depfence

With ML behavioral scoring

The optional ml extra adds scikit-learn-powered behavioral scoring for higher-accuracy obfuscation detection.

bash
pip install "depfence[ml]"

pipx (isolated environment)

bash
pipx install depfence
# or run without installing:
pipx run depfence scan .

uv

bash
uv tool install depfence
# or run ephemerally:
uvx depfence scan .
Offline use: depfence caches all advisory data locally at ~/.depfence/cache. After the first run, behavioral scanners work fully offline. Network scanners degrade gracefully.

Quick start

Run a full scan against your project directory. depfence auto-discovers all supported lockfiles.

bash
# Full scan — all ecosystems, all 30 scanners
depfence scan .

# Fail CI on high+ severity findings
depfence scan . --fail-on high

# Output SARIF for GitHub Code Scanning
depfence scan . --format sarif -o depfence.sarif

# Quick advisory-only scan (no behavioral analysis)
depfence audit .

# Generate a health summary (pass / warn / critical)
depfence summary .

depfence scan

The primary command. Runs all 30 scanners in parallel, aggregates findings, and outputs in the requested format.

bash
depfence scan [PATH] [OPTIONS]

OPTIONS:
  --fail-on     critical|high|medium|low|any|none  (default: none)
  --format      table|json|sarif                   (default: table)
  -o, --output  Path for output file
  --scanner     Comma-separated list of scanners to run (default: all)
  --exclude     Comma-separated scanner names to skip

--fail-on sets the minimum severity that triggers exit code 1. Policy block rules always trigger exit code 1 regardless of this flag.

audit · diff · fix

audit

Advisory-only scan — faster than scan because it skips behavioral analysis. Good for fast CI gates where you want pure CVE coverage.

bash
depfence audit .
depfence audit . --fail-on critical

diff

Delta scan — only checks packages that changed since the last baseline. Ideal for pull request checks where full scans are too slow.

bash
depfence diff .        # delta vs last saved baseline
depfence baseline .    # save current state as baseline

fix

Analyzes findings and generates minimum-bump version upgrades. Preview as a diff or apply directly to lockfiles.

bash
depfence fix .                # preview as diff
depfence fix . --apply         # write changes to manifests
depfence update-plan .        # analyze which updates are safe to merge

sbom · report

sbom

Generates a CycloneDX 1.5 SBOM with full vulnerability enrichment (EPSS, KEV, GHSA). Use --format spdx for SPDX 2.3 output.

bash
depfence sbom . -o sbom.cyclonedx.json
depfence sbom . --format spdx -o sbom.spdx.json
depfence sbom-diff before.json after.json   # compare two SBOMs

report

Full interactive HTML security report — dependency tree, severity breakdown, finding detail with remediation advice.

bash
depfence report . --format html -o report.html
depfence compliance . -f json -o compliance.json

Registry firewall

The registry firewall intercepts npm install and pip install before packages reach disk. It runs depfence checks on each incoming package in real time.

bash
depfence firewall enable .     # install hooks for this project
depfence firewall disable .    # remove hooks
depfence firewall status .     # check hook state

# Pre-check a package before installing
depfence firewall check-pip transformers
depfence firewall check-npm @scope/pkg

When a blocked package is caught at install time:

depfence.yml configuration

Place depfence.yml at your project root. All fields are optional.

yaml
# depfence.yml

# Scanners to run (default: all)
scanners:
  - slopsquat
  - behavioral
  - npm_advisory
  - pypi_advisory
  - osv
  - license_scanner
  - secrets
  - mcp_scanner

# Fail CI on these severities
fail_on: high

# Output format for CI
format: sarif
output: depfence.sarif

# Ecosystems to include (default: auto-detect)
ecosystems:
  - npm
  - pypi
  - cargo

# Cache directory
cache_dir: ~/.depfence/cache

# Ignore specific CVE IDs globally
ignore:
  - GHSA-xxxx-xxxx-xxxx
  - CVE-2024-12345

Policy-as-code

Policy rules evaluate on every scan and can block CI independently of --fail-on. block rules always produce exit code 1. warn rules add findings without blocking.

yaml
rules:
  - name: no-gpl-in-production
    description: Block copyleft-licensed packages
    match:
      license_category: copyleft
    action: block

  - name: require-provenance-for-popular
    description: Require SLSA provenance for high-download packages
    match:
      weekly_downloads_min: 100000
      has_provenance: false
    action: block

  - name: warn-on-ownership-change
    description: Flag packages whose maintainers changed recently
    match:
      ownership_changed_days: 30
    action: warn

  - name: no-install-scripts-npm
    description: Block npm packages with install scripts
    match:
      has_install_scripts: true
    action: block
    ecosystems: [npm]

  - name: allow-internal-registry
    description: Trust packages from the private registry
    match:
      registry: "https://registry.internal.example.com"
    action: allow
Initialize a starter policy: depfence init . scaffolds a depfence.yml with sensible defaults based on your project type.

Inline suppression

Suppress individual findings inline without modifying config files. Use the depfence: ignore comment syntax adjacent to the dependency declaration.

requirements.txt

text
requests==2.28.0  # depfence: ignore CVE-2023-32681 until:2025-12-31
urllib3==1.26.18  # depfence: ignore reason:"internal use only, not internet-facing"

package.json

json
{
  "dependencies": {
    "lodash": "4.17.19"  // depfence: ignore GHSA-p6mc-m468-83gw
  }
}

CLI helper

bash
# Add suppression to a finding interactively
depfence ignore GHSA-xxxx-xxxx-xxxx --reason "false positive in test env" --until 2026-01-01
Warning: Suppressions should include a reason and expiry date. depfence warns when suppressions expire to prevent permanently silenced findings.

GitHub Actions integration

Composite action (recommended)

One line. SARIF results appear in the repository Security tab under Code Scanning alerts.

yaml
- uses: ericrihm/depfence@v1
  with:
    fail-on: high           # critical|high|medium|low|any|none
    format: sarif           # table|json|sarif
    upload-sarif: true      # upload to GitHub Code Scanning

Full workflow

yaml
name: Dependency Security
on:
  push:
    branches: [main]
    paths:
      - '**/package-lock.json'
      - '**/yarn.lock'
      - '**/requirements.txt'
      - '**/poetry.lock'
      - '**/Cargo.lock'
      - '**/go.sum'
  pull_request:
    paths:
      - '**/package-lock.json'
      - '**/yarn.lock'
      - '**/requirements.txt'
      - '**/poetry.lock'
  schedule:
    - cron: '0 6 * * 1'     # weekly advisory refresh

jobs:
  depfence:
    runs-on: ubuntu-latest
    permissions:
      security-events: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install depfence
      - run: depfence scan . --format sarif -o depfence.sarif --fail-on high
      - uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: depfence.sarif
          category: depfence

AI-specific scanning

depfence's AI scanning module covers three distinct threat categories that traditional advisory databases do not address.

Slopsquatting

LLMs regularly suggest package names that don't exist. Attackers monitor AI assistant outputs and register those names within hours of popular queries. The slopsquat scanner scores edit-distance similarity against the 5,000 most-downloaded packages in each ecosystem and flags packages that look like probable hallucination targets.

MCP server security

Model Context Protocol servers execute with broad filesystem and network access. The mcp_scanner audits configs for Claude Desktop, Cursor, VS Code, and Windsurf, looking for hardcoded secrets, pipe-to-shell entrypoints, and entries matching the known-malicious server list. The mcp_fingerprint scanner takes a snapshot of each server's tool schema and alerts when it changes between runs — a rug-pull attack vector.

bash
depfence mcp-scan .           # audit all MCP configs in cwd
depfence mcp-fingerprint .    # snapshot + change detection

ML model supply chain

The model_scanner detects unsafe patterns in Python ML code and model files. It flags torch.load() calls without weights_only=True, scans pickle-format model files for embedded opcodes (OS commands, eval chains), and verifies that HuggingFace model pulls use pinned commit hashes rather than mutable branch refs.

bash
depfence model-scan .         # scan ML code + model files
depfence ai-scan .            # model-scan + AI CVE lookup combined

Pre-commit hook

Add depfence to your pre-commit configuration to scan dependency changes before every commit.

yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/ericrihm/depfence
    rev: v0.4.0
    hooks:
      - id: depfence
        args: ['--fail-on', 'high']
