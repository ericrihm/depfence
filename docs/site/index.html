AI-era threats

The CVE database
doesn't cover AI supply chains

LLMs hallucinate package names. Attackers register them. MCP servers run with broad filesystem access and no auditing. Model weight files in pickle format are arbitrary code execution. None of this shows up in advisory databases.

Read the threat model →
  • ✦
    Slopsquatting & LLM hallucinations Packages hallucinated by ChatGPT, Claude, and Copilot that attackers register within hours. depfence scores edit-distance against the 5,000 most-downloaded packages.
  • ✦
    MCP server rug-pull detection Schema fingerprinting detects when an MCP server silently gains new capabilities after installation — a rug-pull attack. Audits Claude Desktop, Cursor, VS Code, and Windsurf configs.
  • ✦
    ML model supply chain Flags unsafe torch.load calls, detects pickle opcodes in model weight files, and verifies HuggingFace model provenance before weights reach disk.
  • ✦
    Behavioral & network analysis Static AST inspection catches eval-on-download, DNS tunneling, webhook exfiltration, and environment variable theft inside install scripts — before npm install runs.
  • ✦
    Obfuscation detection Base64-exec chains, hex-encoded strings, charcode patterns, and high-entropy payload detection to catch obfuscated malware that bypasses signature scanners.

Quick start

Running in 60 seconds

Install, scan, and ship to CI — no account required, no telemetry, no licensing.

01

Install

Requires Python 3.10+. Runs offline after install.

bash 
pip install depfence
# or with ML behavioral scoring:
pip install "depfence[ml]"
02

Scan your project

Finds lockfiles automatically across all supported ecosystems.

bash 
depfence scan .
depfence scan . --fail-on high
depfence scan . --format sarif
03

Ship to CI

One-line composite action with SARIF upload to GitHub Security tab.

yaml 
- uses: ericrihm/depfence@v1
  with:
    fail-on: high
    upload-sarif: true
bash — common commands 
# Core scanning
depfence scan .                           # full scan, all scanners
depfence audit .                          # advisory-only (fast)
depfence diff .                           # delta scan — only changed packages
depfence fix . --apply                   # auto-fix version bumps

# Specialized scans
depfence mcp-scan .                       # MCP server configs
depfence model-scan .                     # ML model supply chain
depfence gha-scan .                       # GitHub Actions workflows
depfence license-scan .                   # license compliance

# SBOM + reporting
depfence sbom . -o sbom.json             # CycloneDX 1.5
depfence report . --format html          # full HTML report
depfence sbom-diff before.json after.json  # compare two SBOMs

# Registry firewall
depfence firewall enable .                # block at install time
depfence firewall check-pip transformers  # pre-check before install
