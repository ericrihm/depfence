Python API — basic usage

depfence exposes a public Python API via the depfence package. The core entry point is the Orchestrator, which coordinates all scanner execution.

python
import asyncio
from depfence.core.orchestrator import Orchestrator
from depfence.core.models import ScanConfig

async def main():
    config = ScanConfig(
        path=".",
        fail_on="high",
        format="json",
        scanners=None,        # None = all scanners
    )

    orchestrator = Orchestrator(config)
    result = await orchestrator.scan()

    print(f"Findings: {len(result.findings)}")
    for finding in result.findings:
        print(f"  [{finding.severity}] {finding.package} - {finding.title}")

asyncio.run(main())

Running a specific scanner directly

python
from depfence.scanners.slopsquat import SlopsquatScanner
from depfence.core.models import PackageMeta
import asyncio

scanner = SlopsquatScanner()

packages = [
    PackageMeta(name="pytorch-cuda-nightly", version="2.0.0", ecosystem="pypi"),
    PackageMeta(name="langchian", version="0.1.0", ecosystem="pypi"),
]

findings = asyncio.run(scanner.scan(packages))
for f in findings:
    print(f.title, f.severity, f.metadata)

Accessing EPSS and KEV data

python
from depfence.core.epss import EpssClient
from depfence.core.kev import KevClient
import asyncio

async def main():
    epss = EpssClient()
    kev = KevClient()

    # EPSS probability for a CVE (0.0 to 1.0)
    score = await epss.get_score("CVE-2024-3094")
    print(f"EPSS: {score.probability:.4f} ({score.percentile:.1f}th percentile)")

    # Check if CVE is in CISA KEV
    is_kev = await kev.is_known_exploited("CVE-2024-3094")
    print(f"In KEV: {is_kev}")

asyncio.run(main())

Data models

PackageMeta

python
from pydantic import BaseModel

class PackageMeta(BaseModel):
    name: str
    version: str
    ecosystem: str                  # "npm" | "pypi" | "cargo" | "go" | ...
    registry: str | None = None
    lockfile_path: str | None = None
    is_direct: bool = True          # vs transitive dependency
    metadata: dict = {}             # raw registry metadata

Finding

python
class Finding(BaseModel):
    id: str                         # e.g. "GHSA-xxxx-xxxx-xxxx"
    title: str
    severity: Severity              # CRITICAL | HIGH | MEDIUM | LOW | INFO
    scanner: str                    # scanner name
    package: str
    version: str
    ecosystem: str
    description: str
    remediation: str | None = None
    cve_ids: list[str] = []
    cvss_score: float | None = None
    epss_score: float | None = None
    is_kev: bool = False            # in CISA KEV list
    file_path: str | None = None    # for secrets/IaC findings
    line: int | None = None
    metadata: dict = {}             # scanner-specific extras

JSON output

Machine-readable JSON for piping into dashboards or custom automation.

bash
depfence scan . --format json | jq '.findings[] | select(.severity == "CRITICAL")'

# Count critical findings
depfence scan . --format json | jq '.summary.critical'

# Filter by scanner
depfence scan . --format json | jq '[.findings[] | select(.scanner == "slopsquat")]'

Writing a scanner plugin

Plugins are discovered via pip entry points, DEPFENCE_PLUGIN_PATH, or ~/.depfence/plugins/.

python
# mypackage/scanners.py
from depfence.core.models import Finding, PackageMeta, Severity

SUSPICIOUS_TLDS = [".ru", ".cn", ".xyz"]

class TldScanner:
    name = "tld_scanner"
    ecosystems = ["npm", "pypi"]    # or ["*"] for all

    async def scan(self, packages: list[PackageMeta]) -> list[Finding]:
        findings = []
        for pkg in packages:
            homepage = pkg.metadata.get("homepage", "")
            for tld in SUSPICIOUS_TLDS:
                if homepage.endswith(tld):
                    findings.append(Finding(
                        id=f"TLD-{pkg.name}-{pkg.version}",
                        title=f"Suspicious homepage TLD: {tld}",
                        severity=Severity.MEDIUM,
                        scanner=self.name,
                        package=pkg.name,
                        version=pkg.version,
                        ecosystem=pkg.ecosystem,
                        description=f"Package homepage uses {tld} TLD: {homepage}",
                        remediation="Verify package legitimacy before use.",
                    ))
        return findings

Registering a plugin

Register your scanner in pyproject.toml using the depfence.scanners entry point group.

toml
[project.entry-points."depfence.scanners"]
tld_scanner = "mypackage.scanners:TldScanner"
bash
depfence plugins
# Scanners: npm_advisory, pypi_advisory, behavioral ... tld_scanner
Dynamic loading: Set DEPFENCE_PLUGIN_PATH=/path/to/plugins to load plugins from a directory without a pip install. Useful during local development.

Webhook / event hooks

Configure depfence to POST structured JSON events to a webhook URL when findings are produced.

yaml
# depfence.yml
webhook:
  url: https://your-service.example.com/depfence/events
  on:
    - finding.critical
    - finding.high
    - scan.complete
  headers:
    Authorization: "Bearer ${WEBHOOK_TOKEN}"
json
{
  "event": "finding.critical",
  "timestamp": "2025-05-01T12:00:00Z",
  "project": "my-app",
  "finding": {
    "id": "DF-SLOP-001",
    "title": "Slopsquatting - pytorch-cuda-nightly",
    "severity": "CRITICAL",
    "scanner": "slopsquat",
    "package": "pytorch-cuda-nightly",
    "version": "2.0.0"
  }
}

Event bus integration

depfence can emit finding events to any HTTP event bus or webhook endpoint for automated triage, alerting, and cross-repo security posture tracking.

yaml
# depfence.yml
integrations:
  eventbus:
    enabled: true
    url: ${DEPFENCE_EVENT_BUS_URL}
    auth_header: "Bearer ${DEPFENCE_EVENT_BUS_TOKEN}"
    emit_on:
      - finding.critical
      - scan.complete
    tags:
      project: my-app
      env: production
json — event envelope
{
  "source": "depfence",
  "source_version": "0.4.0",
  "event_type": "security.finding",
  "timestamp": "2025-05-01T12:00:00Z",
  "payload": {
    "finding": { "...": "..." },
    "project_tags": { "project": "my-app", "env": "production" },
    "scan_id": "scan-uuid-here"
  }
}
